# Blackbox Android host

Native Kotlin host scaffold that links the shared C ABI from `engine/ffi` via JNI. This directory contains source files to copy into **your** Android app — there is no Gradle project in the repo.

## Build the engine

```bash
node scripts/build-android-aarch64.mjs
```

Produces `dist/android-aarch64/libblackbox_ffi.so`.

The C header is at `engine/ffi/include/blackbox.h` (generated by `cargo build -p blackbox-ffi`).

## Gradle / NDK integration

1. Copy `libblackbox_ffi.so` into your app's `src/main/jniLibs/arm64-v8a/` (or the equivalent path in your module).
2. Build the thin JNI bridge in `apps/android/blackbox/src/main/cpp/` (links the prebuilt Rust `.so`).
3. Add the `blackbox` library sources as an Android library module, or copy them into your app.

`BlackboxEngine.kt` wraps the C ABI through the included JNI bridge.

## Bundle content

```bash
cargo run -p blackbox-bundler --release -- \
  data/silent_archive_game/scenario.json \
  --platform android -o dist/bundle-android
```

Ship `project.box.meta`, `shared.box` + `shared.box.meta`, and the per-chapter box/map pairs. Release
maps may point to `.box.zst` blobs. Parse each box's 16-byte header, load the shared msgpack
catalogs and required `content/chapters/<id>` entries, then pass them to
`BlackboxEngine.create(...)`.

## API

- `BlackboxEngine.initialize()` — once at startup (`bb_init`)
- `BlackboxEngine.create(..., library, randomSeedOverride)` — msgpack content with optional library and RNG seed override
- `getCurrentView()` — revisioned JSON snapshot
- `submitCommand(json, viewRevision)` — revision-checked JSON view delta
- `loadCatalog(bytes)` / `loadLibrary(bytes)`
- `loadChapter(bytes)` / `unloadChapter(chapterId)`
- `serializeState()` / `restoreState(json)` — JSON save and revisioned restore snapshot

Decode JSON into data classes aligned with `apps/web/src/types/game.ts`.
